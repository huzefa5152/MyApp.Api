## TL;DR

The app is well-architected at the seams (JWT validation, CORS, RBAC primitives, audit logging, idempotent migrations, mobile-first compliance) but has **two genuine "act today" exposures** and meaningful **cross-tenant gaps** below that. Architecture is sound but eroding under feature pressure: services are growing too big, controllers are doing data access, and the tenant-access guard is a manual call easy to miss. No silver bullets; everything is fixable.

## 🔴 CRITICAL — rotate / patch today

### #1 ⬜ JWT signing key + Gemini API keys in git history

**Files:** `appsettings.json` (commits b906faa…9208820), `appsettings.Development.json` on disk

The JWT key `HakimiTraders2025SuperSecretKeyForJwtTokenGen!@#$%` was committed for ~10 commits and is recoverable from `git log -p`. Two Gemini keys exposed similarly. **If this key is still in use anywhere (prod), anyone who clones the repo can mint admin tokens for any user.**

**Action:**
- Rotate JWT key in every environment (forces re-login, fine).
- Revoke both Gemini keys at console.cloud.google.com and reissue.
- Optionally `git filter-repo` to scrub history if the repo will go public; otherwise rotate is the only effective fix.

---

### #2 ✅ `ProductImagesController` was unauthenticated — **resolved**

**File:** `Controllers/ProductImagesController.cs:7`
**Commit:** `a1f87fa` (2026-05-05)

Marketing landing page fetches product imagery before login, so the endpoint **should be public**. Was working because the class lacked `[Authorize]`, not because anyone said so.

Resolved by adding explicit `[AllowAnonymous]` + class-level doc comment so the intent is declared and a future "require auth by default" global policy won't silently break the landing page. The controller is correctly bounded:
- Category param is regex-locked to `[a-zA-Z0-9-]+`
- Returns image URLs from `wwwroot/images/products/` only
- Filtered by extension whitelist
- No DB, no tenant data, no PII

---

### #3 ⬜ `LookupController` has zero authorization gates

**File:** `Controllers/LookupController.cs` (entire file)

No `[HasPermission]`, no `[AuthorizeCompany]`, no `_access.AssertAccessAsync`. `POST /api/lookup/items/fbr-defaults` lets any authenticated user mutate the shared catalog. `ItemDescriptions` carries per-tenant FBR defaults today; if it ever becomes tenant-scoped you'll have a painful retrofit.

**Action:** Add `[HasPermission("lookup.view")]` per endpoint; `[Authorize]` at the class minimum.

---

### #4 ⬜ Logo upload accepts arbitrary file extensions

**File:** `Controllers/CompaniesController.cs:121-141`

Reads `file.FileName` and writes `company_{id}{ext}` to `wwwroot/data/uploads/logos/`. No whitelist, no size cap. Operator with `companies.manage.update` can upload `.html` / `.svg`-with-script and serve it. The avatar upload (lines 142-145) gets this right — same guards belong here.

**Action:** Mirror avatar upload — whitelist `.jpg/.jpeg/.png/.webp`, cap 5MB, lowercase the ext.

## 🟠 HIGH — this week

### #5 ⬜ Cross-tenant audit-log exposure

**Files:** `Controllers/AuditLogsController.cs:13-42`, `Middleware/GlobalExceptionMiddleware.cs:142`

Any user with `auditlogs.view` sees every tenant's audit records. `RequestBody` is sensitive-field-redacted (password/token/apikey/secret), but a 4xx on `/api/companies/{id}` for a different tenant still records that company's NTN/CNIC/STRN/address. Stack traces leak server internals to the same audience.

**Action:** Either restrict the permission to seed-admin only, or add `CompanyId` to `AuditLog` and filter through `ICompanyAccessGuard.GetAccessibleCompanyIdsAsync`. Stop persisting `RequestBody` on cross-tenant 4xx at minimum.

---

### #6 ⬜ `_access.AssertAccessAsync` is a manual contract

**Files:** `Controllers/DeliveryChallansController.cs` and 11 others

Of 27 controllers, only 12 reference `ICompanyAccessGuard`. Each call site requires the developer to remember: load entity → look up `CompanyId` → assert. One forgotten call = cross-tenant read. `[AuthorizeCompany]` covers route-param case but ~30+ entity-loaded checks are by hand.

**Action:** Add a global `IAuthorizationFilter` that requires either `[AuthorizeCompany]`, `[AllowAnonymous]`, or a `[BypassCompanyGuard]` opt-out on every action (default-deny). Or push the assertion into base service methods.

---

### #7 ⬜ `/api/auth/me` leaks `SeedAdminUserId`

**File:** `Controllers/AuthController.cs:72-73`

Returns `SeedAdminUserId` to any logged-in user. Combined with #1's historical key leak, an attacker can craft `Sub=1` token. Frontend already has `IsSeedAdmin` boolean.

**Action:** Remove `SeedAdminUserId` from the response.

---

### #8 ⬜ Frontend bundle bloat — full unused UI libraries shipped

**File:** `myapp-frontend/package.json`

`@mui/material`, `@emotion/react`, `@emotion/styled`, `react-bootstrap`, `bootstrap`, `xlsx`, `html2pdf.js` — declared, **zero imports** anywhere in `src/`. Plus `aos`/`swiper`/`react-countup` only used by the public landing page but bundled into the authenticated SPA. Main chunk is 2.3MB.

**Action:** Remove unused deps; `React.lazy` the public landing page so its deps don't ship to operators. Pattern to copy: `utils/exportUtils.js:48-50` already dynamically imports `html2canvas` + `jspdf`.

## ✅ What's solid (keep doing)

### Backend
- **JWT key validation** in `Program.cs:33-40` refuses to boot with a weak key. Excellent fail-loud pattern.
- **CORS is explicit** (`Program.cs:157-176`), no `AllowAnyOrigin`, configurable.
- **No SQL injection surface.** Zero `FromSqlRaw`/`ExecuteSqlRaw` with user input. EF `.Contains()` translates to parameterized LIKE.
- **Bcrypt for passwords**; HSTS + ForwardedHeaders correctly wired.
- **`GlobalExceptionMiddleware`** centralizes error handling and redacts sensitive fields (password/token/apikey/secret/jwt/authorization/connectionstring) before persisting bodies.
- **`PagedResult<T>` + `Pagination:DefaultPageSize`** is the convention everywhere it's used.
- **Idempotent migrations with state-checks** (`AuditLog` markers like `RBAC_USERCOMPANIES_BACKFILL_V1`) — perfect production-safe pattern.
- **Repository interfaces don't expose `IQueryable`** — boundary stays clean.
- **`AuthorizeCompany` + `HasPermission` attribute combo** is the right primitive even though application is uneven (see #6).
- **Login rate-limit** is in place (`Program.cs:64-80`) — 10/min/IP.
- **Avatar upload** properly whitelists extensions, caps at 7MB, uses safe filenames.
- **Excel import** caps file count, per-file size, request size; rejects unsupported extensions.

### Frontend
- **API layer centralized** — `Grep "import axios"` returns ONE hit (`api/httpClient.js:2`); every other module uses the configured client. Cleanest part of the codebase.
- **Permissions are first-class** — `PermissionsContext` exposes `has`/`hasAny`/`hasAll` + declarative `<Can>` gate; pages use both forms.
- **`window.confirm` / `window.alert` ban is intact** — zero matches; `useConfirm()` + `notify()` adopted consistently (52 `notify()` call-sites).
- **Mobile card pattern is real** — 12 surfaces have working `*-cards` ↔ `<table>` swaps under `@media (max-width: 767.98px)`. CSS-only, no JS resize listener.
- **Pagination convention is followed almost everywhere.** `AuditLogsPage`, `StockDashboardPage`, `ItemRateHistoryPage` correctly omit `pageSize` from the request and read it from the response. The two intentional exceptions (`GoodsReceiptForm pageSize:100`, `InvoicePage pageSize:10000`) are commented.
- **Heavy export deps dynamically imported** (`utils/exportUtils.js:48-50` does `await Promise.all([import("html2canvas"), import("jspdf")])`).
- **Three tight contexts** (Auth, Company, Permissions) — no prop-drilling, memoized values via `useMemo`.
